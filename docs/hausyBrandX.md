# Hausy Brand UI Direction

Use this as the visual direction for the Hausy app UI. The goal is to match the design language shown in the reference image: minimal black-and-white foundation, colourful event photography, and very limited purple accents for trust/status moments.

## 1. Overall Style

Hausy should feel like **Hinge for events** — a premium, editorial, social discovery feed for offline gatherings.

The interface should not feel like BookMyShow, a ticketing app, or a loud marketplace. It should feel calm, curated, trustworthy, and adult.

**Design keywords:**

- Premium
- Minimal
- Editorial
- Trustworthy
- Social
- Offline-first
- Calm, not boring
- Fun through photography, not through loud UI colours

## 2. Colour System

The UI should be mostly black, white, and soft grey. Colour should come mainly from the event photos.

### Core Colours

| Role | Colour | Hex | Usage |
|---|---:|---:|---|
| Primary Black | Hausy Black | `#0F0F0F` | Logo, primary text, active tabs, bottom nav active state |
| Background | Hausy White | `#FFFFFF` | Main app background |
| Soft Background | Warm Off White | `#FAFAF7` | Optional page background or card contrast |
| Border Grey | Soft Line Grey | `#E6E6E2` | Card borders, search bar border, dividers |
| Secondary Text | Muted Grey | `#6F6F6F` | Metadata, host labels, inactive nav labels |
| Light Chip | Chip Grey | `#F3F3F0` | Safety tags, category chips, inactive pills |
| Accent Purple | Hausy Purple | `#6B4DFF` | Verification tick, notification dot, tiny brand accent only |

### Accent Rule

Use **Hausy Purple `#6B4DFF` very sparingly**.

Use it only for:

- Verification tick beside host names
- Notification dot
- Tiny logo accent dot
- Small unread/chat indicator
- Rare selected micro-interactions

Do not use purple as a large CTA colour, large background, banner, or heavy UI theme.

## 3. Typography

The Hausy interface uses a mix of a bold modern sans-serif for the logo and clean UI text, with editorial serif typography for event titles.

### Logo Font Style

The `hausy` logo should use a **bold rounded geometric sans-serif** style.

Characteristics:

- Lowercase
- Heavy weight
- Rounded but not childish
- Tight letter spacing
- Black wordmark
- Small purple dot accent near the `y`

Suggested font direction:

- Satoshi Black
- Circular Std Black
- Avenir Next Heavy
- Plus Jakarta Sans ExtraBold
- Inter Tight Black

### UI Font

Use a clean sans-serif for navigation, metadata, chips, and body copy.

Suggested font:

- Satoshi
- Inter
- SF Pro
- Plus Jakarta Sans

### Event Title Font

Event titles should use a high-contrast editorial serif to create a premium, curated feel.

Suggested font direction:

- Playfair Display
- Canela-style serif
- Cormorant Garamond
- Editorial New-style serif
- Libre Baskerville

Use serif typography mainly for:

- Featured event title
- Event card title
- Section hero text if needed

Do not use serif for small UI labels or navigation.

## 4. Layout Style

The home screen should be a vertical event discovery feed.

### Top Header

- Large `hausy` logo at top left
- Location below logo
- Notification icon on top right
- Small purple notification dot
- Plenty of white space

### Search Bar

- Full-width rounded search field
- White or very soft grey background
- Thin grey border
- Placeholder: `Search events, people, or communities`
- Search icon on left
- No heavy fill colour

### Category Tabs

Use simple horizontal pills/text tabs.

Example tabs:

- For You
- Tonight
- New
- Coffee
- Music
- Workshops

Active state:

- Black pill
- White text

Inactive state:

- Text-only or very light divider separation
- Black/grey text
- No colourful tabs

## 5. Event Feed Design

The feed should feel like curated social cards, not ticket listings.

### Featured Card

The first card should be large, cinematic, and photo-led.

Style:

- Large rounded rectangle
- Colourful event photo
- Dark gradient overlay at bottom for readability
- White serif event title
- White metadata
- Host row inside the card
- Attendee avatars inside the card
- Safety chips along the bottom

Important: keep the photo colourful. The UI around it stays black and white.

### Standard Event Cards

Secondary event cards should be white cards with subtle grey borders.

Structure:

- Left image thumbnail
- Right content area
- Serif event title
- Short description
- Date/time and location row
- Host row with verification tick
- Attendee avatars and going count
- Save/bookmark icon on the right
- Safety chips at bottom

## 6. Trust & Safety Chips

Safety chips should look calm and serious, not colourful or playful.

Use:

- White or very light grey fill
- Thin grey border
- Small monochrome icons
- Black or grey text
- Rounded pill shape

Example chips:

- Verified Host
- Women-friendly
- First-timer Friendly
- Balanced Crowd
- Moderated Room

Verification ticks beside host names can use **Hausy Purple `#6B4DFF`**.

## 7. Navigation

Bottom navigation should be minimal and monochrome.

Tabs:

- Home
- Discover
- Saved
- Chats
- Profile

Active state:

- Black icon
- Black label

Inactive state:

- Grey icon
- Grey label

Only use purple for a tiny unread dot on Chats or Notifications.

## 8. Image Direction

Photos are the main source of energy and personality.

Use colourful, warm, real-world images of:

- Small group conversations
- Coffee socials
- Indie music nights
- Workshops
- Rooftop gatherings
- Book clubs
- Creative meetups
- Intimate community events

Avoid:

- Stock-looking party photos
- Large concert crowds
- Overly polished influencer imagery
- Ticketing poster graphics
- Loud red/orange promotional banners

## 9. UI Rules

Follow these rules strictly:

1. Keep the UI mostly black, white, and grey.
2. Let event photography provide colour.
3. Use purple only for verification, notification, and tiny brand accents.
4. Do not use large purple buttons or purple backgrounds.
5. Use serif typography for event titles to create editorial premium feel.
6. Use sans-serif typography for all functional UI.
7. Avoid marketplace-style cards, price-first layouts, and ticketing language.
8. Make the experience feel like browsing social rooms, not booking tickets.
9. Safety should be visible but subtle.
10. The app should feel serious enough to trust and warm enough to join.

## 10. Suggested Copy Tone

Use short, human, offline-first copy.

Examples:

- `Stories Over Strangers`
- `Real stories. Good strangers.`
- `Slow coffee, good conversations.`
- `Live indie. Intimate vibes.`
- `Find plans worth leaving home for.`
- `Verified Host`
- `Women-friendly`
- `First-timer Friendly`
- `Balanced Crowd`

Avoid:

- `Buy tickets now`
- `Book fast`
- `Limited sale`
- `Mega event`
- `Offer ending soon`

## 11. Final Visual Summary

Hausy should look like:

> A premium black-and-white social discovery app where colourful event photography creates warmth, serif titles create editorial elegance, and small purple accents quietly signal verification and notifications.
